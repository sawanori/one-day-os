# 審判通知仕様書 (Judgment Notification Specification)

## 概要

「OSによる審判」は、ユーザーを励ますためのものではありません。資料にある**「鏡（ミラー・テスト）」の概念を応用し、ユーザーが今この瞬間に「理想の自分」から逃げ出していないかを冷徹に暴くための「問い」**です。

---

## 1. 審判（通知）の5つのカテゴリー

AIはユーザーが「マクロ（2.0x）」で設定した今日のクエストと、「広角（0.5x）」にある1年後の目標を照らし合わせ、以下の5つの角度から不意打ちを食らわせます。

| カテゴリー | 通知文の例 | 狙い（何を審判するか） |
|-----------|-----------|---------------------|
| 逃避の検知 | 「今、重要でないふりをしている最重要のことは何か？」 | 雑務に逃げて、本来の苦痛（クエスト）を避けていないか。 |
| 観測者の視線 | 「観察者は今、君を何を望んでいる人間だと結論づけるか？」 | 他人の目（ミラー）を意識させ、今の振る舞いを正させる。 |
| アイデンティティの不一致 | 「今の行動は、君が定義した『理想の自分』と一致しているか？」 | 設定した1.0xのセルフイメージと、今のダラけた姿の乖離を突く。 |
| アンチビジョンの想起 | 「今の1分間は、君を5年後の地獄へ一歩近づけていないか？」 | 怠惰の代償（0.5xの負の未来）を思い出させ、焦りを生む。 |
| 生存確認 | 「沈黙は死だ。君はまだ自分を諦めていないか？」 | 5秒以内のレスポンスを求め、OSへの従属と覚悟を再確認させる。 |

## 3. 「YES/NO」の審判がもたらす結末

ユーザーは通知をタップし、瞬時に答える必要があります。

- **YESを選択:** 「私は今、理想に向かって正しく動いている」という宣言。IHは維持されます。
- **NOを選択:** 「私は今、自分を裏切っている」という自白。IHが大幅に削られ、画面にグリッチノイズが発生します。
- **5秒スルー:** 「OS（構造）の無視」。最も重いペナルティ。IHが激減し、AIから「沈黙は敗北と同義だ」と通告されます。

## 設計メモ

- この通知は、ユーザーの集中を削ぐためのものではない。**「意識のピントを常にマクロ（2.0x）に戻させるための、外側からの衝撃」**
- ランダムにくるからこそ、ユーザーはいつ審判されてもいいように、常に背筋を伸ばして過ごさざるを得なくなる
